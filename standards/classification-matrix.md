# Classification Matrix

**Document ID:** AIWA-STD-03
**Version:** 1.0
**Status:** Active
**Scope:** Mapping of work type to classification category and allowed actions.
**Cross-Reference:** AIWA-DOC-01 (Classification), AIWA-STD-02 (Definitions), AIWA-DOC-03 (Process & Procedures)

---

## 1. Purpose

1.1. This document maps each work type to its permitted classification categories, allowed actions, and AI use eligibility.

1.2. This matrix is a reference tool. It does not replace the full classification process defined in AIWA-DOC-01 and AIWA-DOC-03.

1.3. Classification is assigned on a work-by-work basis. The matrix states what is possible for each work type. The actual classification applied to a specific work depends on the review process.

---

## 2. How to Read This Matrix

- **Work Type** — The formal category of the work as defined in AIWA-DOC-01 Section 3.
- **Permitted Classifications** — The classification categories that may apply to this work type.
- **Monetization** — Whether the work may be monetized under the stated classification.
- **AI Use** — Whether the work may enter an AI system under the stated classification.
- **Conditions** — Any mandatory conditions that apply regardless of the specific classification assigned.

### 3.11. Folklore Classification

*Folklore is a classification category, not a work type. It applies across work types when the conditions for folklore are met. A work classified as folklore is assigned Class 4 or Class 4A.*

| Classification | Monetization | AI Use | Conditions |
|---|---|---|---|
| Class 4 — Restricted Work (Folklore) | Blocked unless specifically approved | Blocked unless lawfully authorized | Community allocation mandatory; held in trust for the people of The Gambia |
| Class 4A — Government-Authorized Restricted (Folklore) | Within verified authorization scope only | Only if authorization explicitly permits | Authorization is scope-limited and does not transfer ownership; community allocation mandatory |

## 4. General Rules

4.1. Classification is assigned on a work-by-work basis through the review process in AIWA-DOC-03.

4.2. AI use is not automatically permitted for any class of work.

4.3. Lexical extraction applies to all work types and all classifications. It is not governed by this matrix.

4.4. Government authorization expands permitted use but does not remove underlying protection.

4.5. When in doubt about classification, apply the higher restriction. The burden of proving a lower restriction falls on the review process.

4.6. Folklore is never Class 1 or Class 2. If a work meets the folklore definition in AIWA-DOC-01 Section 4, it must be classified as Class 4 or Class 4A.

4.7. Community allocation is mandatory for all Class 4 and Class 4A works with communal origin. It is not optional.
